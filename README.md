# simple-ts-app
  
## npm scripts の説明

このプロジェクトで使える npm scripts の内容と使い方は以下の通りです。

### dev
- **コマンド:** `npm run dev`
- **説明:** 開発用サーバーを起動します。ファイルを編集すると自動でリロードされ、ブラウザで動作確認ができます。

### build
- **コマンド:** `npm run build`
- **説明:** TypeScript のコンパイル（`tsc`）を行い、その後 Vite で本番用のビルド（`vite build`）を作成します。本番公開用の最適化されたファイルが生成されます。

### preview
- **コマンド:** `npm run preview`
- **説明:** `build` で作成した本番用ファイルをローカルサーバーで確認できます。本番環境に近い動作をテストできます。

### test
- **コマンド:** `npm test` または `npm run test`
- **説明:** テストを実行します。`vitest` というテストツールで、プロジェクト内のテストコードが動きます。devコマンドと同様に、自動で変更が適用されます。

## 参考

- 環境設定にコストをかけずに、コード(アプリケーションコード・テストコード)に集中するためのライブラリを導入しています。

### vite

- TypeScriptを実行するあれこれを良い感じにやってくれるぐらいの認識で大丈夫です。
- 素のTypescriptで利用する場合は、HTMLのscriptでリンクするのはコンパイル後のjsファイル…などコンパイルを意識する必要がありますが、viteを用いているとscriptにtsファイルを直接指定できます。
  - `npm run build`を行うと、viteがよしなにやってくれた結果のファイルが`dist/`に出力されます
  - jsやcssは最小化され、htmlからはそれらが読み込まれています
- 開発中にコードを変更したら都度コンパイルをする必要もなく、`npm run dev`で変更を自動で適用してくれます。

参考: https://zenn.dev/comm_vue_nuxt/articles/what-is-vite

### vitest

- jestと似た構文でテストを記述できます。
- viteと同じ開発元で、viteと同じように環境のあれこれを良い感じにやってくれるので多くの設定をせずに動きます。
