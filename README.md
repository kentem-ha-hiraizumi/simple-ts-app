# simple-ts-app

## npm scripts の説明

このプロジェクトで使える npm scripts の内容と使い方は以下の通りです。

### dev

- **コマンド:** `npm run dev`
- **説明:** 開発用サーバーを起動します。ファイルを編集すると自動でリロードされ、ブラウザで動作確認ができます。

### build

- **コマンド:** `npm run build`
- **説明:** TypeScript のコンパイル（`tsc`）を行い、その後 Vite で本番用のビルド（`vite build`）を作成します。本番公開用の最適化されたファイルが生成されます。

### preview

- **コマンド:** `npm run preview`
- **説明:** `build` で作成した本番用ファイルをローカルサーバーで確認できます。本番環境に近い動作をテストできます。

### test

- **コマンド:** `npm test` または `npm run test`
- **説明:** テストを実行します。`vitest` というテストツールで、プロジェクト内のテストコードが動きます。dev コマンドと同様に、自動で変更が適用されます。

## 参考

- 環境設定にコストをかけずに、コード(アプリケーションコード・テストコード)に集中するためのライブラリを導入しています。

### vite

- TypeScript を実行するあれこれを良い感じにやってくれるぐらいの認識で大丈夫です。
- 素の Typescript で利用する場合は、HTML の script でリンクするのはコンパイル後の js ファイル…などコンパイルを意識する必要がありますが、vite を用いていると script に ts ファイルを直接指定できます。
  - `npm run build`を行うと、vite がよしなにやってくれた結果のファイルが`dist/`に出力されます
  - js や css は最小化され、html からはそれらが読み込まれています
- 開発中にコードを変更したら都度コンパイルをする必要もなく、`npm run dev`で変更を自動で適用してくれます。

参考: https://zenn.dev/comm_vue_nuxt/articles/what-is-vite

### vitest

- jest と似た構文でテストを記述できます。
- vite と同じ開発元で、vite と同じように環境のあれこれを良い感じにやってくれるので多くの設定をせずに動きます。
